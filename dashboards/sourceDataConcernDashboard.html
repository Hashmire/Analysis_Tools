This dashboard serves as a focal point for CVE Program stakeholders to review how the data they have contributed to CVE records can be refined to improve platform identification automation efforts.

The dashboard analyzes source data contributions and identifies specific concerns that may impact automated processing, helping contributors understand where their data can be optimized for better machine readability.

🏗️ Dashboard Structure

  • Header - Project information and title
  • File Loader - Upload your Source Data Concern Report JSON
  • Overview Statistics - High-level metrics across all sources
  • Source Analysis - Detailed breakdown by data source

🔍 Source Analysis Features

  • Search Field - Find specific sources quickly
  • Interactive Charts - Visual breakdown of concerns vs clean data
  • CVE Records Count - Number of CVE records with source contributions
  • Platform Entries Count - Affected array entries from the source
  • Concern Type Analysis - Categorized breakdown of data quality issues

📊 Understanding Source Cards

  • High-level Metrics - CVE records, platform entries, total concerns
  • Concern Breakdowns - Detailed categorization with subcategories
  • CVE ID Cards - Individual CVE analysis with expandable details
  • Drill-down Links - Click CVE IDs for detailed analysis pages

🎯 How to Use

  • Load Data - Upload a sourceDataConcernReport.json file
  • Review Overview - Check overall statistics in the top cards
  • Find Your Source - Use search to locate relevant sources
  • Analyze Concerns - Click CVE cards to see detailed breakdowns
  • Deep Dive - Click CVE IDs to view full analysis reports
💡 Pro Tip: The Source Analysis section provides the most actionable insights. Each concern type includes subcategories to help you understand the specific data quality improvements that would benefit automation efforts.
